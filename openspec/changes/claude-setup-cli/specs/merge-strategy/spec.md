## ADDED Requirements

### Requirement: Detect existing .claude directory
The system SHALL check for an existing `.claude/` directory before generating files.

#### Scenario: No existing directory
- **WHEN** `.claude/` does not exist
- **THEN** the system SHALL create it and proceed with generation

#### Scenario: Existing directory found
- **WHEN** `.claude/` already exists
- **THEN** the system SHALL prompt the user to choose: merge with existing config or replace entirely

### Requirement: Merge mode
The system SHALL merge generated config with existing files when user selects merge.

#### Scenario: Merge settings.json
- **WHEN** user selects merge and `.claude/settings.json` exists
- **THEN** the system SHALL deep-merge permissions arrays (union), append new hooks (skip duplicates by command string), and merge MCP servers (skip existing keys)

#### Scenario: Merge CLAUDE.md
- **WHEN** user selects merge and `.claude/CLAUDE.md` exists
- **THEN** the system SHALL append new sections to the existing file with a `## Generated by cc-init` header, preserving all existing content above

#### Scenario: Merge agents directory
- **WHEN** user selects merge and `.claude/agents/` exists with files
- **THEN** the system SHALL create only agents that do not already exist (by filename) and SHALL NOT overwrite existing agent files

### Requirement: Replace mode
The system SHALL replace the entire `.claude/` directory when user selects replace.

#### Scenario: Replace with backup
- **WHEN** user selects replace
- **THEN** the system SHALL rename the existing `.claude/` to `.claude.backup.<timestamp>/` before generating new files

### Requirement: Force flag
The system SHALL support a `--force` flag that replaces without prompting.

#### Scenario: Force replace
- **WHEN** user runs `cc-init --force`
- **THEN** the system SHALL back up and replace without prompting for merge/replace choice
